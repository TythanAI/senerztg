# Как продавать эти боты

Короткая инструкция, как передать один магазин клиенту и сколько это стоит.
Полный список всех 50 магазинов с товарами и ценами — в [`CATALOG.md`](CATALOG.md)
(генерируется командой `make catalog`).

## Что именно вы продаёте

Готового Telegram-бота-магазина «под ключ»: каталог, корзина, приём оплаты
(карта + крипта + оплата при получении), автоматическая выдача товара и
админ-панель. Клиенту остаётся вписать токены и залить свой товар — логика,
платежи и защита уже готовы и протестированы.

## Передача одного бота клиенту (5 шагов)

1. Скопируйте проект `bots/` (это движок) — можно целиком или через архив
   (`make archive`).
2. Оставьте нужный конфиг `config/<slug>.yaml`; лишние можно удалить.
3. Отдайте клиенту `env/example.env` и раздел «Развёртывание» из `README.md`.
4. Клиент создаёт бота в @BotFather, подключает платёжки (провайдер и/или
   @CryptoBot), вписывает токены в `env/<slug>.env`.
5. Запуск: `docker compose up -d <slug>` — и магазин работает.

Правки бренда, цен и текстов — прямо в `config/<slug>.yaml` (или массово через
генераторы в `tools/`, затем `make regen`).

## Варианты монетизации

| Модель | Суть | Ориентир |
|--------|------|----------|
| Продажа исходника | Отдаёте бота + конфиг клиенту навсегда | разово |
| Настройка «под ключ» | Разворачиваете на VPS клиента, подключаете платёжки | разово + за настройку |
| Аренда / подписка | Хостите у себя, берёте помесячно | ежемесячно |
| Пакет ниш | Несколько магазинов одному клиенту со скидкой | пакетом |

Себестоимость запуска низкая: все 50 ботов помещаются на одном VPS
(1–2 vCPU / 1–2 ГБ ОЗУ), общий образ, отдельная БД у каждого.

## Чек-лист перед передачей

- [ ] Токен бота (@BotFather) — свой для каждого клиента.
- [ ] Платёжка подключена: `PROVIDER_TOKEN` (карта) и/или `CRYPTOBOT_TOKEN` (крипта).
      Для физических магазинов достаточно оплаты при получении.
- [ ] `ADMIN_IDS` — Telegram id владельца (кнопка `/admin`).
- [ ] Товар залит через `/admin → Пополнить склад` (для цифровых).
- [ ] Цены и тексты проверены в `config/<slug>.yaml`.
- [ ] `python tools/selfcheck.py` — зелёный.

## Полезные команды

```bash
make catalog     # обновить CATALOG.md по всем конфигам
make regen       # пересобрать все конфиги и docker-compose.yml
make selfcheck   # оффлайн-проверка (без токенов и сети)
make archive     # собрать zip-архив всего проекта (для флешки/передачи)
```

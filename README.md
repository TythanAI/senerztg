# senerztg — монитор скинов CS2

Telegram-бот, который следит за Торговой площадкой Steam и мгновенно присылает
уведомления, когда:

- 🎯 продают скин с **редким паттерном** (blue gem и т.п.), **низким/экстремальным
  флоатом** или ценными **наклейками** (Katowice 2014 ищется на всех лотах) —
  и цена при этом близка к обычной;
- 🚀 цена скина **резко скакнула** (+12% за 30 минут по умолчанию).

## Быстрый старт

```bash
pip install -r requirements.txt
export BOT_TOKEN="токен_от_@BotFather"   # Windows: set BOT_TOKEN=...
python skin_monitor_bot.py
```

Уведомления приходят в Telegram на ID `1164070668` (поменять — `ADMIN_ID`
в начале `skin_monitor_bot.py`).

**Команды бота:** `/status` — что мониторится, `/last` — последние 10 событий.

## Как добавить свой скин

В `watchlist.json`, по образцу:

```json
{
  "name": "M9 Bayonet | Case Hardened (Field-Tested)",
  "paint_seeds": [601],
  "max_float": null, "min_float": null,
  "sticker_keywords": [], "max_overpay_pct": 20
}
```

`name` — это market_hash_name: возьми его из адресной строки на странице
предмета в маркете (после `/730/`, декодируй %7C → `|` и т.д. — проще всего
скопировать точное название предмета).

## Важно знать (честно)

- **«Мониторить вообще всё» нельзя.** На маркете десятки тысяч предметов, а
  Steam банит IP за частые запросы. Потолок для одного бесплатного IP —
  ~15–20 предметов с паузами. Поэтому в списке — предметы, где редкие
  паттерны реально стоят денег.
- **Снайпить против профи не выйдет.** Грубо недооценённые лоты выкупают
  коммерческие боты за доли секунды. Наша ниша — лоты, которые висят
  минуты/часы: продавец не знал про свой паттерн и поставил обычную цену.
- **Буст ≠ сигнал покупать.** Комиссия маркета 15%: рост меньше неё — убыток.
- Данные — публичные эндпоинты маркета + бесплатный API csfloat.com.
  Никакого доступа к твоему аккаунту Steam боту не нужно.

## Автозапуск по расписанию (GitHub Actions, бесплатно)

В `.github/workflows/monitor.yml` уже настроен запуск каждые ~5 минут.
Чтобы заработало:

1. **Задай секрет с токеном:** репозиторий → Settings → Secrets and variables
   → Actions → New repository secret → имя `BOT_TOKEN`, значение — токен от
   @BotFather. Без него воркфлоу упадёт с понятной ошибкой в логах.
2. Вкладка **Actions** → включи воркфлоу, если попросит → «steam-monitor» →
   Run workflow (первый запуск вручную). Дальше сам по расписанию.

**Проверка жизни.** При первом успешном запуске бот пишет «✅ Монитор
подключён и работает». Дальше молчит, пока не найдёт что-то интересное —
это норма (редкие лоты появляются раз в часы/дни). Раз в сутки шлёт «💓
жив». Если Steam не отвечает раннерам GitHub — пришлёт предупреждение и
подскажет проверить логи.

> ⚠️ Расписание в GitHub Actions не гарантирует точность: реально проходы
> идут раз в 5–20 минут (в часы пик реже). Для паттернов и наклеек этого
> хватает; для настоящего реалтайма используй постоянный `skin_monitor_bot.py`.

## Если Steam блокирует серверы GitHub (HTTP 403/429)

Steam часто режет запросы с IP дата-центров. Тогда монитор пришлёт «⚠️ Steam
блокирует запросы...» и не найдёт ничего. Решение — гонять тот же
`cron_check.py` по расписанию с домашнего компьютера (у него обычный IP):

**Windows (Планировщик заданий):** создай задачу, которая каждые 5 минут
запускает `python cron_check.py` в папке проекта, с переменной окружения
`BOT_TOKEN`. **Linux/Mac (cron):**

```cron
*/5 * * * * cd /путь/к/senerztg && BOT_TOKEN=твой_токен /usr/bin/python3 cron_check.py
```

Код один и тот же — просто исполняется на твоём IP, а не на серверах GitHub.
Состояние копится в `state.json` локально.
